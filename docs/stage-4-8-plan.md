# DirectPilot AI — план реализации этапов 4–8

## Этап 4. AI-рекомендации в безопасном режиме
- Формировать рекомендации только как черновик (draft).
- В каждом ответе указывать risk + expected impact + next_step.
- Обязательная пометка: рекомендации не применяются напрямую.

## Этап 5. Preview + Approval
- На preview считать policy-нарушения и risk_score.
- На approval переносить policy verdict в карточку согласования.
- Блокировать авто-применение без статуса approved.

## Этап 6. Журнал действий
- Логировать создание preview, запрос согласования, approve/reject.
- Хранить risk_score и policy_violations для разборов.

## Этап 7. Оценка качества ИИ
- Добавить offline evaluation script с кейсами.
- Метрика первого уровня: pass-rate по ключевым требованиям (конверсии, CPA, approval).

## Этап 8. MVP для пилотов
- Чеклист запуска на 3-5 агентств:
  1. Подключение OAuth Яндекс.
  2. Валидация целей Метрики.
  3. Первая неделя: только preview + ручное approval.
  4. Вторая неделя: включение ограниченного auto-режима для low-risk рекомендаций.
